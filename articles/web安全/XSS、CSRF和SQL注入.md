# CSRF和SQL注入

## 二、CSRF

cross site request forgery，跨站请求伪造。如：攻击者诱使用户访问一个页面，就以该用户的身份在第三方站点里执行了一次操作。

**本质：重要操作的所有参数可以被攻击者猜测到。**

#### 危险性

* 利用用户身份操作用户账号

#### 防御

防御的方案是针对“重要操作的所有参数可以被攻击者猜测到”，构建一个不能被猜到的参数，比如：

* **token**
* 验证码（不能加太多验证码，只能当做辅助操作）
* referer check（服务器并非任何时候都能取得referer）
